Tracarbon: Track Your Device's Energy Consumption and Calculate Carbon Emissions

Overview

Tracarbon is a Python library designed to track device energy consumption and calculate carbon emissions based on real-time data. It offers developers a simple yet powerful toolset to monitor and analyze the environmental impact of their software applications. By leveraging system-level metrics and integrating with environmental data sources, Tracarbon provides insights into the carbon footprint associated with computational tasks.

Key Features

  • Real-Time Monitoring: Track energy consumption metrics in real-time using system-level interfaces like RAPL (Running Average Power Limit).
  • Carbon Emission Calculations: Calculate carbon emissions associated with energy consumption using live CO2 intensity data from sources like CO2Signal.
  • Integration: Easily integrate Tracarbon into existing Python projects to monitor and analyze energy usage patterns.

Purpose

The primary purpose of Tracarbon is to empower developers and organizations to make informed decisions regarding energy-efficient software development. By quantifying the environmental impact of computational tasks, Tracarbon enables proactive measures to reduce carbon emissions and promote sustainable computing practices.

Why It's Useful

  • Environmental Accountability: Provides developers with the tools to understand and mitigate the environmental impact of their code.
  • Optimization Insights: Identifies energy-intensive processes and encourages optimization strategies for improved efficiency.
  • Compliance and Reporting: Supports compliance with environmental regulations and facilitates reporting on carbon footprint metrics.

Tracarbon strives to bridge the gap between software development and environmental responsibility, fostering a more sustainable approach to technology innovation.

Install Tracarbon

You can install Tracarbon and optional exporters using pip. Here are the installation commands:

Install Tracarbon

Install the core Tracarbon library:

pip install tracarbon

Install Optional Exporters

Tracarbon supports various exporters like Datadog, Prometheus, and Kubernetes. You can install them along with Tracarbon using extras:

pip install 'tracarbon[datadog,prometheus,kubernetes]'

Exporters

Exporters in Tracarbon allow you to send your energy consumption and carbon emission metrics to various output formats and monitoring tools. Here's a detailed look at the available exporters:

Stdout

  • Description: Prints the metrics directly to the standard output (console).
  • Use Case: Useful for quick debugging and local development to see the metrics immediately without any additional setup.
  • Configuration: No additional configuration is required. Simply run Tracarbon, and the metrics will be printed to the console.

JSON

  • Description: Writes the metrics to a JSON file.
  • Use Case: Ideal for scenarios where you need to store the metrics in a structured format for further analysis or processing.
  • Configuration: Specify the file path where the JSON file should be saved.

Prometheus

  • Description: Sends the metrics to a Prometheus server.
  • Use Case: Suitable for environments where Prometheus is used for monitoring and alerting. This allows you to integrate your device's energy consumption and carbon emission metrics into your existing monitoring setup.
  • Configuration: Specify the URL of the Prometheus Push Gateway.

Datadog

  • Description: Sends the metrics to Datadog.
  • Use Case: Useful for users who leverage Datadog for infrastructure monitoring and performance metrics. This exporter allows you to visualize and analyze your energy consumption data alongside other metrics in Datadog.
  • Configuration:
    • Datadog API Key: Set the Datadog API key as an environment variable.
    • Datadog Site: Specify the Datadog site (datadoghq.com).

Usage

Tracarbon provides flexibility in how you can track device energy consumption and calculate carbon emissions. You can use it via the command line interface (CLI) or integrate it programmatically into your Python projects.

Command-Line Interface (CLI)

  • Basic Usage
    • Run Tracarbon CLI with default settings:

      To start tracking energy consumption and carbon emissions using the default Stdout exporter and CO2 Signal API (requires API key):

      TRACARBON_CO2SIGNAL_API_KEY=API_KEY tracarbon run
    • Run Tracarbon CLI without the CO2 Signal API:

      To run Tracarbon with the default stdout exporter, which prints metrics directly to the console, without requiring a CO2 Signal API key:

      tracarbon run
    • Specify a location with country code (ISO 3166-1 alpha-2 format):

      To specify a location other than the default global setting:

      tracarbon run --country-code-alpha-iso-2 fr

      Replace fr with the desired country code.

    • Use a specific exporter like Datadog:

      To send metrics to Datadog, configure Tracarbon with Datadog API and application keys:

      TRACARBON_CO2SIGNAL_API_KEY=API_KEY DATADOG_API_KEY=DATADOG_API_KEY DATADOG_APP_KEY=DATADOG_APP_KEY tracarbon run --exporter-name Datadog

      Adjust with your actual API and app keys.

    • Running Tracarbon CLI on Linux Hardware with Kubernetes, Sending Metrics to Prometheus:

      To run Tracarbon CLI on Linux hardware within a Kubernetes environment and send the metrics to Prometheus:

      tracarbon run --exporter-name Prometheus --containers
  • Programmatically Integrate Tracarbon
    • Using Tracarbon in Python projects:

      Import Tracarbon modules and configure it programmatically:

      from tracarbon import TracarbonBuilder, TracarbonConfiguration

configuration = TracarbonConfiguration(co2signal_api_key="API_KEY")  # Your configuration
tracarbon = TracarbonBuilder(configuration=configuration).build()
tracarbon.start()
# Your code
tracarbon.stop()

with tracarbon:
# Your code

report = tracarbon.report # Get the report
    • Custom Metrics and Exporters:

      Customize Tracarbon by defining custom metrics and exporters:

      from tracarbon import TracarbonBuilder, TracarbonConfiguration
from tracarbon.exporters import StdoutExporter
from tracarbon.general_metrics import CarbonEmissionGenerator, EnergyConsumptionGenerator

configuration = TracarbonConfiguration(co2signal_api_key="API_KEY")  # Your configuration
metric_generators = [EnergyConsumptionGenerator(), CarbonEmissionGenerator()]
exporter = StdoutExporter(metric_generators=metric_generators) # Your exporter
tracarbon = TracarbonBuilder(configuration=configuration).with_exporter(exporter=exporter).build()
tracarbon.start()
# Your code
tracarbon.stop()

with tracarbon:
# Your code

report = tracarbon.report # Get the report
    • Custom Configuration:

      Create a custom configuration for specific needs:

      from tracarbon import TracarbonBuilder, TracarbonConfiguration

configuration = TracarbonConfiguration(co2signal_api_key="API_KEY")  # Your configuration
